---
name: Enfoque MVP estricto
description: No agregar complejidad innecesaria, respetar fases, no sobre-engineerear
type: feedback
---

Respetar siempre el enfoque MVP. No agregar complejidad innecesaria.

**Why:** El usuario quiere construir por fases y validar antes de escalar. Experiencias previas con sobre-ingenieria causan friccion.

**How to apply:** No proponer features fuera del scope actual. No agregar automatizaciones, multi-tenancy, ni abstracciones prematuras. Si algo se puede hacer simple, hacerlo simple. Preguntar antes de agregar algo que no fue pedido.
